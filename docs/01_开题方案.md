## 【第2页】执行摘要

**本页结论：金策将分散金融信息转化为可追溯、可解释、可审核、可执行的投研结论与财富管理方案，区别于一切"黑箱生成"类工具。**

- **做什么**：基于多智能体、可信证据链、因果推演、组合分析和合规审查的人机协同金融决策系统。业务闭环：多源信息接入 → 任务拆解 → 证据检索校验 → 多智能体分析 → 因果链与情景推演 → 组合影响评估 → 客户画像与财富适配 → 合规审查 → 人工确认 → 报告输出 → 全程审计追溯。
- **为什么现在**：中国个人可投资资产278万亿元（2022末，招行×贝恩2023），券商财富管理转型"三分投、七分顾"成行业共识；至少16家券商已本地化部署DeepSeek用于投研（新浪财经/每经2025-02）；但通用大模型+RAG在金融问答中81%答错或拒答（FinanceBench，arXiv:2311.11944），"可信"是落地最大缺口。
- **给谁用**：券商/期货/基金/资管研究员、投资经理、财富顾问、风控合规人员；经B2B2C触达专业投资者。
- **凭什么做成**：四类先导项目（知识管理、多模型MoA网关、个人工作台、金融交易工具链）已提供智能体编排、证据留痕、工作流状态机与真实金融场景经验（详见作品集）。
- **边界**：AI仅辅助研究与辅助决策，不承诺收益、不自动荐股、不替客户决策，重要结论附证据、风险提示、适当性检查与审计记录。

## 【第3页】对金仕达命题的理解

**本页结论：金仕达的核心资产是"交易、风控、合规、机构客户"四件事，金策的落地位置是叠加其上的"可信智能决策入口"。**

金仕达1995年成立、源于复旦大学，服务400多家中资金融机构（证券、期货、银行、资管），业务覆盖三大板块：金融市场交易管理、金融风险管理、大宗商品/产业数字化（金仕达官方公众号/媒体报道）。

关键产品线与本方案的对应关系：

| 金仕达能力 | 对金策的意义 |
|---|---|
| 期货柜台交易系统（V6.0/V8T/DTP-F核心交易系统，信创测评通过） | 交易执行与行情数据的潜在对接层（后续拟接入） |
| KGo多元资管系统 | 组合与财富管理模块的机构落点 |
| 易经金综合金融终端/智能策略交易系统 | 机构投研助手的终端载体 |
| CTRM大宗商品贸易与风险管理、HMS套期保值管理（2022证券期货科学技术奖二等奖） | 产业客户套保场景的智能分析入口 |
| 帝喾大模型（因果AI+隐私计算，集成五大金融终端，用于智能风控） | 因果推演与风控理念同源，可协同 |
| 合规大模型（法律法规库、智能合规问答与监测；2025人工智能大模型金融应用大赛"卓越奖"） | "合规即流程"的企业侧能力基础 |

本方案理解命题的关键：金仕达不需要再造一个聊天机器人，而需要一个**与其交易、资管、套保、合规系统协同的可信决策层**——这决定金策以证据链、合规内嵌、私有化部署为设计原点（结合点为推断，落地口径见第21页）。

## 【第4页】行业现状

**本页结论：市场足够大、转型方向确定、大模型已规模化进场，但"可信与合规"仍是行业空白带。**

- **市场体量**：中国个人可投资资产2022年末278万亿元，为全球第二大财富管理市场（招商银行×贝恩《2023中国私人财富报告》）；麦肯锡预计2025年个人金融资产达332万亿元。
- **转型方向**：券商财富管理从"卖产品"转向"买方投顾"，"三分投、七分顾"成为行业共识（麦肯锡）；基金投顾试点2019-10启动，约60家机构获批。
- **大模型落地事实**：同花顺问财HithinkGPT于2024-01发布，为业内首家通过网信办备案的金融对话大模型；恒生电子LightGPT（2023-06）及"光子"投研应用；东方财富"妙想"；蚂蚁AntFinGLM"支小助/支小宝"（2023-09外滩大会）；度小满开源"轩辕XuanYuan"（2023-05）；2025-02至少16家券商完成DeepSeek-R1/V3本地化部署，用于投研与合规问答（新浪财经/每经2025-02）。
- **效率缺口**：知识型员工约19%工时用于检索搜集信息（麦肯锡全球研究院，非投研专属，证据强度中）——金融投研的信息密度更高，该缺口只会更大。
- **结构性空白**：现有终端重数据、轻证据；通用大模型重生成、轻校验；合规工具与投研流程割裂。可信证据链与合规内嵌的一体化平台尚缺位。

## 【第5页】用户真实痛点

**本页结论：用户的痛不是"没有AI"，而是"AI给的答案不敢用、不能用、出了事说不清"。**

基于公开研究与产品观察（竞品用户观点见研究底稿，均为公开渠道用户反馈，非本项目访谈）：

1. **不敢用——幻觉与无据结论**：GPT-4-Turbo+RAG在150道金融问答中81%答错或拒答，幻觉与无法从长财报取证是主因（FinanceBench，arXiv:2311.11944）；共性缺陷包括数字计算错误、数据过时、无可点击引用、把观点包装成事实。
2. **不能用——责任与合规风险**：Air Canada案（2024 BCCRT 149）中客服机器人编造退款政策，公司被判须兑现——机构须为AI输出担责；投资咨询属特许业务，无牌照荐股违法，2024-10监管专项整治"非法荐股"（含AI生成荐股内容）。
3. **说不清——流程割裂**：研究员在终端、文档、聊天工具间手工搬运；事实、市场共识、模型推断混在同一段文字里；结论无法回溯到来源与时间点。
4. **到不了客户——投研与财富断层**：研究结论停留在内部报告，缺少从"宏观事件→资产影响→客户组合→适当性匹配→客户可懂材料"的连续链路。
5. **工具体验差**：国际终端价格极高（Bloomberg约27,660美元/用户/年）、本土产品模块收费复杂、私有化与审计能力不足（公开用户反馈）。

## 【第6页】目标用户

**本页结论：先服务机构专业用户，再经B2B2C触达高净值客户——买单方与合规责任方都在机构内部。**

**第一优先级（机构内部，直接用户）：**

| 角色 | 核心任务 | 金策给什么 |
|---|---|---|
| 券商/期货/基金/资管研究员 | 事件跟踪、证据检索、报告撰写 | 证据原生的研究流水线，产出可引用、可回溯 |
| 投资经理与自营人员 | 情景判断、组合调整 | 因果链+三情景推演+组合风险敞口量化 |
| 财富顾问 | 客户沟通、方案适配 | 客户画像匹配、客户版材料、适当性前置校验 |
| 风控与合规审核人员 | 内容审查、留痕审计 | 合规即流程：自动检测+人工审批+全程审计 |

**第二优先级（间接受益）**：高净值客户、专业投资者、经B2B2C模式获得机构服务的个人客户——他们看到的是"客户版报告"，决策与责任仍在持牌机构一侧。

**非目标用户**：寻求"AI直接告诉买什么"的零售投机者——金策不提供、也明确拒绝此类用法。

## 【第7页】用户旅程

**本页结论：同一事件，研究员走"证据链"，财富顾问走"适配链"，两条链在合规与人工确认处汇合。**

**旅程A：宏观研究员（内部研究链路）**

1. 输入任务："评估美联储进一步加息信号对主要资产的影响" → Research Planner拆解为数据、宏观、资产、情景子任务
2. 证据检索：央行讲话原文、行情、历史数据自动带来源、发布时间、原文位置入库
3. 事实/观点/推断分层标注，冲突数据触发人工确认点
4. 因果链构建（利率→美元→实际利率→黄金/权益/汇率），Causal Analyst给出失效条件
5. 三情景推演（乐观/基准/悲观）+ 历史相似事件对照
6. Evidence Auditor逐条校验证据支持率 → Report Integrator生成内部报告
7. 人工确认 → 发布，全程审计留痕

**旅程B：财富顾问（客户适配链路）**

1. 收到研究结论推送 → 导入客户持仓（模拟数据）与风险画像
2. Portfolio Analyst测算组合风险暴露与压力测试结果
3. Wealth Advisor Agent按风险等级、期限、流动性约束生成多备选方案
4. Compliance Reviewer做适当性匹配、收益承诺检测、风险提示完整性检查
5. 财富顾问人工确认（责任节点） → 生成客户版报告（事实与建议分离、AI内容标识）
6. 客户沟通后记录留痕，可审计追溯

## 【第8页】旗舰应用场景

**本页结论：一个"美联储加息"事件跑通全部六大子系统，证明平台不是功能拼盘而是一条完整流水线。**

**场景**：美联储释放进一步加息信号 → 分析对黄金、原油、美股、人民币资产及某模拟客户组合的影响 → 生成内部研究报告与客户沟通材料。（客户信息与持仓均为**模拟数据**。）

**流程演示**：

| 步骤 | 动作 | 体现的能力 |
|---|---|---|
| 1 | 检索央行讲话原文、新闻、宏观数据、行情 | 多源接入+来源标记 |
| 2 | 事实、市场共识、争议观点、模型推断分层 | 事实观点推断分层 |
| 3 | 构建因果链：政策利率→美元指数→实际利率→黄金；利率→折现率→美股估值；利差→人民币资产 | 因果推演 |
| 4 | 乐观/基准/悲观三情景+失效条件+历史相似事件 | 情景推演 |
| 5 | 模拟客户组合：黄金/权益/汇率敞口测算、压力测试 | 组合影响评估 |
| 6 | 按客户风险等级/期限/流动性给备选方案 | 财富适配 |
| 7 | 合规审查：适当性、收益承诺、风险提示完整性 | 合规即流程 |
| 8 | 人工确认 → 内部版+客户版双份报告 → 证据链与审计留痕 | 人机协同 |

**演示边界**：所有结论标注"模型推断"或"情景假设"，不含任何收益承诺；客户版报告含AI内容标识与完整风险提示。

## 【第9页】产品定位

**本页结论：金策是"人机协同的可信决策基础设施"，用排除法划清与五类工具的边界。**

**金策是什么**：面向研究员、投资经理、财富顾问及风控合规人员的人机协同金融决策系统——AI负责检索、校验、推演、起草与初审，人负责判断、确认与责任。

**金策不是什么（明确排除）**：

| 不是 | 区别 |
|---|---|
| 普通财经聊天机器人 | 每条结论挂证据链，可点击回溯原文；无证据不输出结论 |
| 新闻摘要工具 | 做因果推演与组合测算，不止于信息压缩 |
| 自动研报生成器 | 事实/观点/推断分层+审计留痕，非一次性生成 |
| 自动荐股系统 | 不输出个股买卖指令；投资咨询属特许业务，无牌照荐股违法（2024-10监管专项整治含AI荐股） |
| 替代持牌投顾的自动决策工具 | 重要建议保留人工确认节点，责任在持牌机构 |

**七大创新点**：证据原生；事实观点推断分层；因果推演与失效条件；投研到财富管理一体化；合规即流程；多智能体交叉审核；人机协同。大模型、RAG、多智能体只是实现手段，不作为创新点本身。

## 【第10页】整体解决方案：六层架构

**本页结论：六层子系统自下而上构成"数据可信→分析可信→结论可信→流程可信"的递进结构。**

| 层 | 子系统 | 关键职责 |
|---|---|---|
| L1 | 多源金融数据与知识层 | 行情、宏观、政策、公告、财报、新闻、研报、内部材料、客户持仓、风险画像接入；实体识别、时间有效性、来源标记、冲突检测、原文定位、知识图谱、向量检索+结构化查询；权限分级与脱敏 |
| L2 | 多智能体投研引擎 | 11个角色分工协作、交叉审核（详见第12页） |
| L3 | 可信证据链 | 来源、时间、原文位置、事实依据、推断摘要、关键假设、反例、置信度、失效条件十要素（详见第14页） |
| L4 | 因果分析与情景推演 | 传导路径、影响方向与时间尺度、关键变量、失效条件、三情景、历史相似事件 |
| L5 | 组合与财富管理 | 持仓导入、资产识别、集中度、风险敞口、压力测试、归因、风险等级匹配、期限与流动性约束、多方案比较、客户版报告 |
| L6 | 合规、风控与AI治理 | 适当性、收益承诺检测、无依据建议识别、风险提示完整性、权限、脱敏、人工审批、AI内容标识、操作日志、模型调用记录、提示词版本、审计追溯、私有化部署、模型回退 |

**设计原则**：L6不是外挂审查，而是贯穿L1-L5的流程内机制；任何一层证据不足即触发回退或人工确认，而非强行生成。

## 【第11页】核心功能模块

**本页结论：八个功能模块对应用户旅程的每一步，每个模块都产出"可审计的中间件"而非黑箱结果。**

| 模块 | 功能 | 可审计产出 |
|---|---|---|
| M1 研究任务台 | 自然语言下达研究任务，Planner拆解为子任务并分派 | 任务拆解树、子任务依赖图 |
| M2 证据检索与校验 | 多源检索、来源与时间有效性校验、冲突检测 | 证据卡片（含原文位置与置信度） |
| M3 事实分层视图 | 已确认事实/市场共识/争议观点/模型推断/情景假设/行动建议/风险提示七类分层展示 | 分层标注记录 |
| M4 因果链工作台 | 传导路径可视化、关键变量、失效条件编辑 | 因果链图谱及版本历史 |
| M5 情景推演器 | 乐观/基准/悲观三情景、历史相似事件对照 | 情景假设清单 |
| M6 组合分析 | 持仓导入（模拟数据）、敞口、集中度、压力测试、归因 | 组合风险报告 |
| M7 财富适配与客户材料 | 风险等级/期限/流动性匹配，多方案比较，生成客户版报告 | 适当性匹配记录、客户版报告 |
| M8 合规与审计中心 | 自动合规检测、人工审批流、AI内容标识、操作与模型调用日志、提示词版本管理 | 审计追溯台账 |

## 【第12页】多智能体架构：11个角色

**本页结论：11个角色以"分工+交叉审核+证据守门"组织，任何结论至少经过一次异角色校验。**

| 角色 | 输入 | 任务 | 输出 | 交叉审核关系 |
|---|---|---|---|---|
| 1 Research Planner | 用户研究任务 | 拆解任务、分派、编排流程 | 子任务树、执行计划 | 计划由Report Integrator复核完整性 |
| 2 Data Analyst | 子任务、数据源 | 数据提取、计算、质量检查 | 结构化数据+质量标记 | 数字结果由Evidence Auditor抽样回查 |
| 3 Macro Analyst | 宏观数据、政策原文 | 政策与宏观环境解读 | 宏观判断（标注推断） | 与Fundamental Analyst互相质证 |
| 4 Fundamental Analyst | 财报、公告、行业数据 | 资产基本面分析 | 基本面结论+依据 | 与Macro Analyst互相质证 |
| 5 Causal Analyst | 各方结论、历史案例 | 构建因果链、给出失效条件 | 因果图谱 | 因果链由Evidence Auditor逐环节核证 |
| 6 Scenario Analyst | 因果链、关键变量 | 三情景推演、历史相似事件对照 | 情景假设清单 | 假设需Causal Analyst确认不自相矛盾 |
| 7 Portfolio Analyst | 情景、客户持仓（模拟） | 敞口、集中度、压力测试、归因 | 组合影响评估 | 测算口径由Data Analyst复核 |
| 8 Wealth Advisor Agent | 组合评估、客户画像 | 风险等级/期限/流动性匹配，多方案比较 | 备选方案（非指令） | 方案须经Compliance Reviewer全检 |
| 9 Compliance Reviewer | 全部产出 | 适当性、收益承诺、无依据建议、风险提示完整性检测 | 合规意见（通过/驳回） | 驳回即回退，具一票否决权 |
| 10 Evidence Auditor | 全部结论与证据 | 逐条核验证据支持率、标记无据内容 | 证据审计报告 | 证据不足可中止流程（守门员） |
| 11 Report Integrator | 审核通过的素材 | 生成内部版/客户版报告、AI内容标识 | 双版报告 | 终稿由人工确认后发布 |

**工作流**：Planner编排 → 分析角色并行 → Causal/Scenario串行深化 → Portfolio/Wealth适配 → Evidence Auditor与Compliance双重守门 → Integrator成稿 → 人工确认。

**异常回退**：证据检索失败→标记"数据缺失"并降级为定性结论（禁止编造）；角色输出冲突→升级至Evidence Auditor仲裁，仍冲突则触发人工确认点；模型调用失败→自动切换备用模型（模型回退机制），全程记录。

**冲突处理**：分析角色间结论冲突不取平均、不掩盖，而是并列呈现争议双方及各自证据，由人裁决。

**人工确认机制**：三处强制人工节点——关键数据冲突时、对外客户材料发布前、合规驳回后的处置决定。

## 【第13页】数据与知识架构

**本页结论：数据架构的第一性原理是"每个事实都知道自己从哪来、什么时间有效、谁能看"。**

**数据源分层**：

| 类别 | 内容 | 接入方式 |
|---|---|---|
| 市场数据 | 行情、宏观指标、利率汇率 | 公开数据源/交易所授权数据（后续拟接入） |
| 基本面数据 | 财报、公告、招股书 | 公开披露渠道 |
| 文本资料 | 新闻、政策原文、研报、内部材料 | 公开新闻源+机构内部授权材料 |
| 客户数据 | 持仓、风险画像、期限与流动性约束 | 机构系统导入（演示全部为**模拟数据**） |

**知识处理流水线**：采集 → 实体识别（公司/人物/指标/事件）→ 来源标记与发布时间/更新时间登记 → 时间有效性管理（过期数据自动降权）→ 冲突检测（同源异值触发告警）→ 原文定位（段落级坐标）→ 入库。

**三类存储协同**：知识图谱（实体与关系，支撑因果链）+ 向量库（语义检索）+ 结构化库（指标与行情），查询时三路召回、Evidence Auditor做一致性校验。

**权限与脱敏**：按角色分级授权；客户数据展示层脱敏；原始数据不出域；为私有化部署预留架构（满足数据分类分级要求，证监会令第218号）。

## 【第14页】可信证据链

**本页结论：证据不是生成后补的引用，而是从数据到建议全链路原生携带的结构化对象。**

**证据十要素（每条结论必带）**：

1. 来源（机构/作者/链接或系统内坐标）
2. 发布时间
3. 更新时间
4. 原文位置（段落级，可点击回溯）
5. 事实依据（支撑该结论的原始材料）
6. 推断摘要（从依据到结论的推理链）
7. 关键假设
8. 反例（已知不支持该结论的证据）
9. 置信度（高/中/低+理由）
10. 失效条件（什么情况下结论不再成立）

**界面七类分层（视觉与交互强区分）**：

| 类型 | 展示规则 |
|---|---|
| 已确认事实 | 有权威来源+时间有效，默认样式 |
| 市场共识 | 多来源一致但非官方，标注"共识" |
| 争议观点 | 并列呈现双方及证据 |
| 模型推断 | 显著标注"AI推断"+置信度 |
| 情景假设 | 标注"假设"+失效条件 |
| 行动建议 | 须经合规检查与人工确认后方可展示 |
| 风险提示 | 强制展示，不可折叠 |

**证据原生 vs 事后引用**：传统生成式工具先成文再找引用，导致引用错位；金策的结论对象在创建时即绑定证据对象，Evidence Auditor可逐条机器校验"结论是否被其证据支持"。

## 【第15页】因果分析和情景推演

**本页结论：把"相关性叙述"升级为"带失效条件的因果链"，让结论可证伪、可监控。**

**因果链构建**（以加息场景为例）：

政策利率上行 → 美元指数走强 → 以美元计价的黄金承压（实际利率通道：名义利率−通胀预期）→ 成长股折现率上升、美股估值承压 → 中美利差变化 → 人民币资产资金流动压力。

每个环节标注：传导机制、影响方向、时间尺度（即时/数周/数季）、关键变量、失效条件（如"若通胀预期同步上行，实际利率未必上升，黄金承压逻辑减弱"）。

**三情景推演**：

| 情景 | 触发假设 | 推演重点 |
|---|---|---|
| 乐观 | 加息落地但指引转鸽 | 风险资产修复、美元回落 |
| 基准 | 加息一次且指引中性 | 黄金区间震荡、美股结构分化 |
| 悲观 | 连续加息+缩表加速 | 美元流动性收紧、新兴市场承压 |

**历史相似事件对照**：检索历史加息周期中同类资产的实际表现作为参照系（标注历史区间与样本局限，历史不代表未来）。

**推演纪律**：情景输出一律标注"情景假设"，含概率判断时标注为"模型推断"；失效条件进入监控清单，条件触发时系统主动提示结论需重估——这是"可证伪"在产品上的实现。

## 【第16页】组合及财富管理

**本页结论：把研究结论翻译为"这个客户的组合受什么影响、有哪些合规备选方案"，打通投研到财富的最后一公里。**

**组合分析能力**：

- 持仓导入与资产识别（演示使用**模拟数据**）：识别个股/基金/期货/现金类资产
- 集中度分析：单一资产、行业、币种集中度
- 风险敞口测算：利率、汇率、权益、商品敞口与情景联动
- 压力测试：将第15页三情景映射为组合损益区间
- 归因分析：收益来源拆解

**财富适配流程**：

1. 客户画像读取：风险等级、投资期限、流动性约束
2. 适当性匹配前置：产品风险等级与客户风险等级比对（证监会令第130号要求"适当的产品给适当的投资者"）
3. 多方案比较：2-3个备选方案并列，各自标注调整逻辑、预期风险特征变化与不确定性（**不承诺收益**）
4. 生成客户版报告：事实与建议分离、AI内容标识、完整风险提示、语言去专业化

**责任边界**：方案为"备选"而非"指令"；财富顾问人工确认后方可对客户出示；客户决策记录留痕——投顾责任始终在持牌机构与持牌人员一侧。

## 【第17页】合规风控和AI治理

**本页结论：合规不是上线前的检查项，而是写进工作流每一步的运行时机制，并对标四部监管文件。**

| 监管要求 | 金策对应机制 |
|---|---|
| 《证券期货投资者适当性管理办法》（证监会令第130号）：了解投资者、产品分级、风险揭示留痕、禁止向不匹配投资者主动推介 | 客户画像强制读取；风险等级硬性匹配；不匹配方案禁止生成；推介行为留痕 |
| 《生成式人工智能服务管理暂行办法》（网信办等七部门令第15号）：备案/安全评估、保护用户输入、内容标识 | 客户版内容强制AI标识；用户输入不用于外部训练；私有化部署支持备案与安全评估 |
| 《证券期货业网络和信息安全管理办法》（证监会令第218号）：数据分类分级、委托处理安全责任 | 数据分级存储、按角色授权、脱敏展示；企业授权后集成的委托处理边界清晰 |
| 《人工智能算法金融应用评价规范》（JR/T 0221—2021，人民银行金融行业标准） | 算法可解释（证据链）、可控（模型回退）、可审计（全程留痕）对标评价维度 |

**AI治理运行时机制**：收益承诺检测（敏感表述自动拦截）；无依据建议识别（Evidence Auditor联动）；权限分级与最小授权；操作日志、模型调用记录、提示词版本全量留存；模型回退（主模型异常时切换备用模型并记录）；审计追溯台账支持事后逐条复盘。

**红线自检**：不输出个股买卖指令；不向不匹配投资者生成推介内容；不虚构数据；AI输出机构担责（Air Canada案，2024 BCCRT 149）→ 因此重要结论必须人工确认。

## 【第18页】产品原型

**本页结论：六张界面覆盖"任务—证据—推演—组合—合规—报告"全流程；以下为产品原型（界面为设计描述，标注"产品原型"）。**

**界面1 研究任务台**（产品原型）：顶部自然语言任务输入框；左侧子任务拆解树实时展开，各节点显示负责智能体与状态；右侧为执行时间线。人工确认点以醒目色块标示。

**界面2 证据工作区**（产品原型）：三栏布局——检索结果列表、证据卡片（十要素完整展示）、原文对照视图（段落级高亮，点击证据即跳转原文位置）；冲突证据并排标红。

**界面3 因果链画布**（产品原型）：节点图可视化传导路径，每条边标注方向、时间尺度与失效条件；右侧面板展示三情景切换与历史相似事件对照。

**界面4 组合分析页**（产品原型）：持仓结构图（标注"模拟数据"水印）、敞口雷达图、三情景压力测试结果区间条、归因瀑布图。

**界面5 合规审查中心**（产品原型）：待审队列、自动检测项清单（适当性/收益承诺/风险提示完整性逐项打勾或驳回）、驳回意见与回退记录、审批流。

**界面6 报告输出页**（产品原型）：内部版/客户版双标签切换；七类分层以不同视觉样式呈现；页脚固定风险提示与AI内容标识；"人工确认发布"按钮为唯一发布入口。

## 【第24页】总结

**本页结论：金策以"可信"为支点，把大模型能力转化为金融机构敢用、能用、可审计的投研与财富决策流程。**

1. **问题真实**：278万亿元财富管理市场（招行×贝恩2023）拥抱AI，但81%的金融问答错误率（FinanceBench，arXiv:2311.11944）与机构担责判例（Air Canada，2024 BCCRT 149）证明"黑箱生成"走不通。
2. **方案对症**：证据原生、七类分层、因果推演与失效条件、投研到财富一体化、合规即流程、多智能体交叉审核、人机协同——七大设计全部指向"可追溯、可解释、可审核、可执行"。
3. **边界清晰**：不是聊天机器人、不是荐股系统；AI辅助决策，人承担责任；不承诺收益、不虚构数据、重要结论人工确认。
4. **落地务实**：与金仕达交易、资管、套保、合规大模型能力互补，仅以"可对接/拟接入/企业授权后集成"口径规划，私有化部署预留。
5. **能力可信**：四类先导项目的真实仓库、部署与文档支撑关键技术路径；全部量化指标以"MVP目标值，待验证"诚实标注。
