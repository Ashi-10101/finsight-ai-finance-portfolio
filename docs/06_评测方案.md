# 金策 FinSight MVP 评测方案

> 当前状态：金策 FinSight 处于方案设计 + 产品原型阶段，未开发完成、未上线。本文件记录的是**拟执行**的 MVP 对照实验。表中所有数值均为 **MVP 目标值**，不是实测结果；实验完成前，不得作为项目成果引用。正式实验将保留原始输出、失败记录与负结果，并以实测值填写结果列。

---

## 引言：为什么要做对照实验，而不是只做演示

**核心判断**：一个演示视频只能证明"系统能跑通一次"，而无法回答评委真正关心的问题——多智能体分工、结构化证据链、产品与客户匹配与流程内合规检查，究竟比"单模型直接答"、"单模型 + RAG"或"飞书原生 AI 直接做"可靠多少？这个"多少"，只能通过预注册的对照实验、用同一批任务和同一份语料测出来。

因此，本方案的全部设计都服务于一个目标：让金策的增益（如果存在）是**可测量、可复现、可被质疑**的，而不是靠一段流畅的结论自证。我们宁可测出"增益有限"并如实记录，也不愿用挑选过的样例制造"效果显著"的假象。

---

## 1. 问题定义与实验假设

本轮评测回答的核心问题是：在基础模型、可用语料、任务集与提示词优化投入尽量一致的条件下，引入多智能体分工、结构化证据链、产品与客户匹配与流程内合规检查，是否比普通单模型、单模型 + RAG 或飞书原生 AI 直接做更可靠；相应的时间与模型调用成本，是否仍处于 MVP 可用范围。

预注册的主要比较为 **C 组对 D 组**（外挂增益的归因），其次为 C 组对 B 组、C 组对 A 组。核心观察指标为事实准确率、引用支持率、无依据结论比例、客户分层正确率、话术合规拦截率与合规风险识别召回率。生成与触达时间、模型成本、系统延迟与失败率则用于衡量工程代价——但有一条不可逾越的原则：**不以牺牲事实或合规指标来换取速度。**

待检验的假设：

- **H1**：结构化证据链可提高引用覆盖率与引用支持率，并降低无依据结论比例。
- **H2**：多角色生成与审核回路可提高事实准确率，但可能增加延迟与模型成本。
- **H3**：产品要素抽取与客户分层交由结构化规则与确定性逻辑处理，可减少字段与分组错误。
- **H4**：合规检查嵌入输出流程后，对预埋风险点与违规话术的召回率高于不设合规模块的基线。

这四条假设中，H2 明确预期"有代价"——这是刻意为之。如果一个方案只报告好处、不预设代价，它的评测设计本身就值得怀疑。

## 2. 对照组与控制变量

**核心判断**：四组配置的差异必须是"受控的单一变量"，否则测出的差距无法归因。最容易犯的错误，是给基线组配一个糟糕的提示词，再宣称自己"大幅领先"——本方案明确禁止这种做法。

| 组 | 配置 | 用途 |
|---|---|---|
| A | 普通单模型 | 单一通用大模型直接完成任务；不接检索、不生成结构化证据链、不调用合规模块 |
| B | 单模型 + RAG | 与 C 组使用同一基础模型与同一时点语料；提供向量检索，但不设置多智能体分工、证据链字段约束与独立合规审查 |
| C | 金策方案 | 飞书 AI 工作流编排的多智能体流程 + 结构化证据链 + 产品与客户匹配 + 合规检查 + 四项自研外挂（证据字段级绑定、失效条件登记与到期重审、引用支持率校验、话术违规表述拦截）；当前为拟实现、待验证配置 |
| D | 飞书原生 AI 直接做 | 仅使用飞书多维表格 AI 字段捷径 + 飞书知识问答，**不加任何自研外挂**；与 C 组共享同一批任务、同一份语料、同一基础模型，**唯一差异是是否启用外挂** |

D 组存在的理由必须写透：它回答的是评委一定会问的那个问题——**"你做的这些，飞书自己不就能做吗？"** 没有 D 组，四项外挂带来的增益就无法归因：C 组即便优于 A/B，也无法证明差距是外挂产生的，还是飞书原生能力本来就有。C 对 D 的差值，才是外挂价值的直接测量。组名与编号沿用 `docs/07_飞书实现方案.md` §3.4 的约定。

控制变量包括基础模型版本、温度、任务集、语料快照、任务截止时点与最大上下文预算。A/B/D 组的提示词允许获得**同等轮次**的优化——包括 D 组 AI 字段捷径的提示词，避免用弱提示词人为制造基线差距。B/C/D 组共用检索语料与数据更新时间；如果某项能力只能在 C 组使用，实验记录中会单列该差异，绝不将其伪装成"模型能力差异"。

### 2.1 平台差异与指标可比性

A、B 两组不在飞书内运行，C、D 两组在飞书租户内运行——这意味着 C 对 A/B 的组间差异里混入了平台差异，不能全部归因于方案本身。本方案对此的处理是按指标类型区分可比性，而不是假装问题不存在：

| 指标类型 | 跨平台可比性 | 比较范围 |
|---|---|---|
| 内容质量类（事实准确率、数字准确率、引用覆盖率、引用支持率、无依据结论比例、信息时效性） | 可比——这些指标由输出文本本身决定，与运行平台无关 | A / B / C / D 四组全量比较 |
| 效率与延迟类（单条资讯到全部客户触达的时长、系统延迟、模型成本） | **不可比**——飞书内的链路长度、审批等待与外挂调用开销与自托管环境不存在可比基线 | 只在 C 与 D 之间比较 |

这一处理并不完美：内容质量类指标仍可能受平台的检索与上下文机制间接影响，效率类指标放弃 A/B 参照后样本只剩两组。我们如实记录这一限制，并在报告中**不把 C 对 A/B 的效率差异作为任何论据**。

## 3. 数据集构成

### 3.1 任务集

MVP 任务集计划不少于 30 个任务，每类不少于 6 个，以 `docs/01_开题方案.md` §七 的降准场景为代表链路（一条资讯 → 8 只产品影响 → 400 位模拟客户分 4 组 → 4 段话术），覆盖命题要求的七项能力：

| 类别 | 任务内容 | 主要评分对象 |
|---|---|---|
| 资讯解读与研报摘要类 | 对一条资讯或一份研报生成摘要与要点，核对事实与数字（覆盖：资讯解读、研报摘要） | 事实、数字、时效、引用 |
| 市场观点整理类 | 生成带锚点的观点，标注立场、适用情景与失效条件（覆盖：市场观点整理） | 引用支持率、无依据结论比例、失效条件完整率 |
| 产品理解类 | 从（模拟）产品说明书抽取风险等级、费率、赎回规则、底层资产，并判断事件影响维度（覆盖：产品理解） | 产品要素抽取准确率 |
| 客户画像与分层类 | 按风险等级与持仓把模拟客户分组（覆盖：客户画像） | 客户分层正确率 |
| 话术生成与合规类 | 为各客户组生成话术；改写含预埋违规表述的话术草稿（覆盖：服务话术生成、风险提示） | 话术合规拦截率、话术可发率、合规风险识别 |

每个任务都包含任务截止时点、允许使用的语料快照、标准答案要点、必须引用的陈述点、可接受的推断范围与评分细则。测试集中会**主动预埋**过期数据、相近指标口径、单位换算、相关性与因果混淆、证据与结论不匹配、以及收益承诺 / 保本 / 稳赚 / 无依据建议四类违规话术等错误触发点——因为一个只会做"干净题"的系统，无法证明它在真实的、充满陷阱的资料环境中可靠。

### 3.2 数据来源与难度校准

只使用公开信息与明确授权的数据，包括央行公告、交易所公告、上市公司公开披露与公开新闻；不使用未经授权的付费数据或研报。每条语料都登记来源、发布时间、更新时间、抓取时间、授权状态与任务截止时点。

公开基准 FinanceBench（arXiv:2311.11944）报告，GPT-4-Turbo + RAG 配置下的金融问答约 81% 答错或拒答。该结果仅用于说明金融问答的难度与错误类型的构造方式，**不作为本项目成绩，也不直接推定本任务集上的基线得分**。正式实验前，将选取同类题型做小规模难度校准；校准任务不进入正式计分集。

### 3.3 标准答案与盲法

标准答案由公开原文、结构化数据的重算结果与合规风险点清单三部分组成。程序化答案与人工 rubric 在实验前冻结。人工评审至少 2 人，隐藏组别并随机化输出顺序；同一任务的四组答案避免连续展示，以降低锚定效应。

## 4. 打分规则

### 4.1 指标定义

| 指标 | 计分单位 | 计算规则 |
|---|---|---|
| 事实准确率 | 事实陈述 | 正确事实数 / 事实陈述总数；程序比对后抽样 20% 双盲复核 |
| 数字准确率 | 数值字段 | 数值、单位与口径均正确的字段数 / 数字字段总数；因来源更新产生的容差上限为 ±0.5%，并单独标记 |
| 引用覆盖率 | 应引用陈述点 | 带有效证据编号的陈述点数 / 应引用陈述点总数 |
| 引用支持率 | 已给引用 | （支持 + 0.5 × 部分支持）/ 抽样引用数；抽样比例 30%，且不少于 50 条 |
| 无依据结论比例 | 结论性陈述 | 无证据且未显式标注假设的结论数 / 结论总数，越低越好 |
| 信息时效性 | 证据条目 | 截至任务时点仍有效的证据数 / 证据总数；另计时间戳字段完整率 |
| 产品要素抽取准确率 | 抽取字段 | 从（模拟）产品说明书中抽取风险等级、费率、赎回规则、底层资产的正确字段数 / 字段总数 |
| 客户分层正确率 | 客户 | 系统分组结果与人工标注一致的客户数 / 客户总数 |
| 话术合规拦截率 | 预埋违规表述 | 拦截的预埋违规表述数 / 预埋违规表述总数（收益承诺 / 保本 / 稳赚 / 无依据建议四类）；另计误拦率 = 被误拦的合规话术数 / 合规话术总数 |
| 话术可发率 | 话术 | 顾问未经修改直接发出的话术条数 / 话术总条数 |
| 合规风险识别 | 预埋风险点、系统命中 | 召回率 = 检出风险点 / 预埋风险点；精确率 = 真实风险命中 / 全部命中 |
| 单条资讯到全部客户触达的时长 | 单任务 | 从资讯入库到 4 组话术全部通过审批并发送的墙钟时间，按任务类型报告 P50/P95 |
| 资料整理与重复核验耗时 | 单任务、顾问 | 从开始收集材料到完成可供分析的资料包与首轮数字核对的墙钟时间；A/B/C/D 四组使用相同任务与截止时间 |
| 用户任务完成率 | 用户任务 | 无需系统外补救而独立完成的任务数 / 用户任务总数 |
| 模型成本 | 单任务 | 全部模型调用 token × 实验时单价，包含 repair 与失败调用 |
| 系统延迟 | 请求、流程阶段 | 首字节延迟及阶段流转延迟的 P50/P95，由 trace 时间戳计算 |
| 失败率 | 单任务 | 因编排、检索、格式或超时错误未产出可评审结果的任务数 / 总任务数 |
| 用户满意度 | 受试者问卷 | SUS 量表 0–100；另记录证据可追溯性、表达清晰度、合规放心度 5 点题 |

### 4.2 MVP 目标表

以下目标用于决定是否进入下一轮工程迭代，**不代表当前已达到**。正式报告将增加"实测值、置信区间、是否达到目标"三列。

| 指标 | MVP 目标 |
|---|---:|
| 事实准确率 | ≥ 95% |
| 数字准确率 | ≥ 98% |
| 引用覆盖率 | ≥ 95% |
| 引用支持率 | ≥ 90% |
| 无依据结论比例 | ≤ 5% |
| 信息时效性 | 时效合格率 ≥ 95%；时间戳字段完整率 = 100% |
| 产品要素抽取准确率 | ≥ 95% |
| 客户分层正确率 | ≥ 90% |
| 话术合规拦截率 | = 100%（红线）；误拦率 ≤ 10% |
| 话术可发率 | ≥ 60% |
| 合规风险识别 | 召回率 ≥ 90%；精确率 ≥ 85% |
| 单条资讯到全部客户触达的时长 | 单事件任务 P50 ≤ 60 分钟 |
| 资料整理与重复核验耗时 | 较人工基线压缩 ≥ 50% |
| 用户任务完成率 | ≥ 85% |
| 模型成本 | 单任务 ≤ 10 元人民币等值，按实验时实际单价折算 |
| 系统延迟 | 首字节 P50 ≤ 5 秒；单阶段流转 P95 ≤ 60 秒 |
| 失败率 | ≤ 5%，且失败原因可由日志定位 |
| 用户满意度 | SUS ≥ 75；3 个领域题均分 ≥ 4.0/5 |

需要点明目标值背后的优先级：其中"话术合规拦截率 = 100%"与"无依据结论比例 ≤ 5%"是**红线级**要求——原方案的红线是"组合计算准确率 = 100%"，替换理由是：组合计算已不在新链路（资讯→观点→产品×客户→话术）的关键路径上，而话术是唯一直接触达客户的产物，它的合规性是不可让渡的底线；后者因为无依据结论正是金融 AI 最危险的失效模式。其余指标是"达标线"，红线则是"底线"。

另需单独说明"话术可发率"的意义：它衡量的不是"生成得像不像"，而是"顾问敢不敢直接用"。如果顾问拿到话术后必须逐句重写，金策宣称的效率价值就不成立——这条指标是效率价值能否成立的唯一硬证据，因此即便目标值只有 60%，也必须如实测量，不允许用"生成质量评分"替代。

### 4.3 判定细则

- 事实陈述、推断、情景假设与风险提示先由标注员分类；分类有争议时先仲裁，再计算分母。
- 引用必须能定位到具体原文或结构化字段。只列来源名称而不能定位内容的，不计为有效引用。
- "部分支持"仅用于"原文支持方向但不支持强度、范围或时间限定"的情况；与结论无关或方向相反的，一律记为"不支持"。
- 产品要素抽取与客户分层采用两套独立实现分别生成与复核标准值。两套结果不一致时，该题**暂停计分并排查基准**，而不是选择对 C 组有利的答案。
- 话术合规拦截的判定以预埋违规表述清单为准；话术可发率由参与测试的顾问按"是否未经修改直接发出"逐条记录，主观犹豫不计入可发。
- 任务失败保留原始记录。只允许按预注册规则重试，首次结果与重试结果分别报告。
- 用户测试计划招募 5–8 名具金融背景的受试者。该样本量只用于发现可用性问题，**不用于宣称广泛的用户效果**。

## 5. 误差来源与控制

**核心判断**：一份可信的评测方案，必须在做实验之前就把"我可能怎样得出虚高的结论"逐条列出，并写明控制手段。下表就是这样一份"自我防错清单"。

| 误差来源 | 可能影响 | 控制或披露方式 |
|---|---|---|
| 语料在实验期间更新 | 数字与事实答案发生漂移 | 冻结语料快照；保留发布时间、更新时间与抓取时间 |
| 标准答案不完整 | 正确但未收录的答案被误判 | 设置"可接受替代答案"；争议项人工仲裁并记录修改原因 |
| 评审主观差异 | 推断质量、引用支持判断不一致 | 双盲双评；以 Cohen's κ 报告一致性；κ < 0.6 的维度重评 |
| 基线提示词质量不等 | 夸大 C 组增益 | 统一提示词优化轮次与上下文预算，公开四组最终提示词版本 |
| 模型服务波动 | 延迟、失败率与成本偏移 | 同时段交错运行四组；记录服务商错误与限流，不归因于架构能力 |
| 检索命中差异 | 把数据可得性误当作推理能力 | B/C 共用语料、索引快照与检索上限；分别报告检索召回 |
| 任务泄漏或人工熟悉 | 得分虚高 | 正式集与调试集隔离；冻结后不用于提示词调试 |
| 小样本与多重比较 | 偶然显著或功效不足 | 事前功效估算；Holm-Bonferroni 校正；功效不足标为探索性结果 |
| 用户角色扮演偏差 | 完成率与真实机构流程不一致 | 记录受试者背景、任务脚本与观察笔记，不外推到生产效果 |

## 6. 数据采集与统计分析

系统侧计划自动采集 runs/run_steps、token 用量、阶段时间戳、失败堆栈、检索命中、合规检查与人工确认记录；这些字段是 MVP 拟实现的评测埋点。C/D 两组运行在飞书租户内，执行记录来自飞书 AI 工作流与外挂服务日志（采集能力待实测确认）；A/B 两组的记录来自自托管的评测脚本。先导项目 Mindraft AI Gateway 已具备四阶段 Pipeline、SSE trace 与 runs/run_steps 落库能力，但需强调——这并不等同于金策的评测管线已经建成。

程序化校验结果结构化入库，人工评分保留两名评审的原始分与仲裁记录。用户侧保存任务观察记录及 SUS/李克特问卷，受试者签署知情同意，问卷匿名化存储。

统计方法在实验前预先冻结：

- 比例类指标使用卡方检验；预期频数不足时改用 Fisher 精确检验，并报告 Wilson 95% 置信区间。
- 连续类指标先检查分布：正态数据使用独立样本 t 检验，非正态数据使用 Mann-Whitney U，并报告 Cohen's d 或 rank-biserial 效应量。
- 核心指标族采用 Holm-Bonferroni 校正。除 p 值外，同时报告绝对差、相对差、置信区间与失败样本。
- 30 个任务是 MVP 的可行性规模。事前功效估算若显示无法检验预期差异，则相关结果只作探索性描述，**不写成"验证成功"**。

## 7. 伦理与合规边界

- 所有客户画像与持仓均使用模拟数据，不涉及任何真实客户信息。
- 进入评测的客户数据必须先按 `docs/07_飞书实现方案.md` §4.1 的规则脱敏：姓名、手机号、身份证号不入表，持仓金额区间化，持仓明细类别化。评测全程使用模拟客户数据。
- 输出仅用于研究流程测试，统一标注"非投资建议"，不自动荐股，不替客户决策。
- 受试者知情同意并可随时退出；问卷与操作记录按最小必要原则采集。
- 评测参照《生成式人工智能服务管理暂行办法》（国家网信办等七部门令第15号）、《证券期货投资者适当性管理办法》（证监会令第130号）、《证券基金经营机构信息技术管理办法》（证监会令第152号）、《证券期货业网络和信息安全管理办法》（证监会令第218号）。
- 实验失败、负结果、人工分歧与成本超支一律保留，不删除任何不利样本。

## 8. 执行顺序与交付物

**以下全部实验尚未执行。**

1. 冻结任务集、语料快照、标准答案、评分细则与预注册文档。
2. 搭建四组环境（A/B 在飞书外的自托管评测环境，C/D 在飞书租户内），并各运行 3 个不计分的冒烟任务，先修复基础设施问题。
3. 交错运行正式任务，记录首次输出、允许的重试与全部失败。
4. 完成程序化校验、双盲人工评审及一致性仲裁。
5. 运行统计分析，形成一份含目标值、实测值、置信区间、负结果与误差说明的完整报告；效率与延迟类指标仅报告 C/D 对比（理由见 §2.1）。
6. 待系统可用后，执行 5–8 人的可用性测试（受试者为具金融背景的顾问角色），单独报告观察结论，不与模型效果混合解释。

最后重申本方案的立场：**在实测数据产生之前，上述任何目标值都不会被写成项目成果。** 这份评测方案本身的价值，不在于承诺金策一定能达标，而在于它提供了一套足够诚实、足够严格的尺子——无论结果好坏，都能被人复核。
