## 这个仓库是什么

金策目前处于**方案设计和产品原型阶段**。这个仓库用于集中展示我们的开题方案、行业研究、界面原型、架构设计、评测计划和报告样稿。

项目想解决的问题来自我平时使用行情软件、财经资讯平台和交易复盘工具时的一个具体感受：资料并不少，但把一项判断从原文出处核对到组合影响，再整理成能给客户看的材料，仍然需要在多个工具之间反复切换。直接使用通用大模型又会遇到时间点不明、数字出处缺失和推断写得过于肯定等问题。

因此，我们没有把金策设计成一个直接回答“买什么”的聊天工具。原型围绕一项研究任务展开：

1. 记录任务范围、数据截止时间和可用来源；
2. 将检索、数据处理、宏观分析、情景分析等工作分开执行；
3. 让重要结论保留原文位置、假设和不成立的条件；
4. 把结论放入模拟客户组合中做压力测试；
5. 在生成客户材料前检查适当性、收益承诺和风险提示；
6. 由研究员、投资经理或财富顾问确认后再输出。

这些步骤目前主要通过设计稿、HTML原型和架构文档表达，尚未开发成可供金融机构使用的完整系统，也没有接入金仕达的生产环境。

## 演示任务

我们先选择了一个范围可控的宏观场景：

> 美联储释放进一步加息信号后，分析黄金、原油、美股和人民币资产可能受到的影响，并查看一个模拟客户组合在不同情景下的风险变化。

这个场景能同时展示原文检索、因果路径、情景假设、组合计算、客户表达和合规检查。页面中的客户、持仓、行情和压力测试数字均为模拟数据，不构成投资建议。

## 和已有项目的关系

金策不是以下项目的改名版本。这些项目彼此独立，只说明我此前做过相关的工程工作。

### Mindraft AI Gateway

Phase 1 MVP已完成并上线自用。实现了Planner、Worker、Reviewer、Integrator四阶段编排，保存run和step记录，并通过SSE展示执行进度。当前完整仓库为Private。

技术栈：Next.js 15、Fastify、Drizzle、Postgres、SSE、Zod。

### Mindraft OS

个人工作台，多模块已投入日常使用。包含日历、项目、服务状态和期货仓位计算器。AI Studio只是界面预览，不代表已经接入真实智能体遥测。当前完整仓库为Private。

### Trader Terminal

本地单用户交易记录工具，v0.1已完成。交易记录按DRAFT、PLANNED、ACTIVE、CLOSED、JOURNALED状态流转；进入计划状态后锁定关键字段，用于约束临时修改交易计划。

技术栈：Next.js 14、tRPC、Drizzle、SQLite、Zod。

### Memomeld

知识管理项目，仍在开发。工程骨架、Notes类型和校验层已经完成并部署，核心笔记业务逻辑尚未完成。当前完整仓库为Private。

## 使用边界

- 金策不提供自动荐股，不承诺收益，也不代替持牌投顾或投资经理作决定；
- 原型中的客户与持仓全部是模拟数据；
- 项目尚未接入金仕达或其他金融机构的生产系统；
- 评测目标需要在MVP完成后通过对照实验验证；
- 公开仓库不包含API密钥、账户信息、服务器地址、交易记录或其他私人数据。

本仓库的材料会继续随MVP开发更新。先导项目在公开前会另建经过检查的展示版本，不会直接将现有私人仓库转为Public。
