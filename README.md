## 这个仓库是什么

金策目前仍处于**方案设计与产品原型阶段**。本仓库用于公开展示项目方案、行业研究、产品原型、技术架构、评测计划及相关文档，作为本次参赛项目的展示仓库，**不包含可直接部署的生产系统**。

金策的设计，来源于我们在使用行情终端、财经资讯平台与交易复盘工具时的真实体验：信息本身并不缺乏，但要完成一次完整的投研分析，往往仍需在多个平台之间反复切换，分别完成资料检索、数据核对、逻辑分析、组合评估与材料整理。

而如果直接依赖通用大模型来做分析，又容易出现四类问题——引用来源不清晰、数据截止时间不明确、数字依据缺失、结论表达过于确定。这些都不符合金融研究与财富管理场景对**可追溯性**与**合规性**的基本要求。

因此，金策没有被设计成一个直接回答"买什么"的聊天工具，而是尝试把一次完整的研究任务，拆分为若干**可追溯、可审核**的工作环节：

1. 记录研究任务范围、数据截止时间及可用数据来源；
2. 将检索、数据处理、宏观分析、情景分析等环节拆分执行；
3. 为关键结论保留原文引用、前提假设与失效条件；
4. 将分析结果放入模拟客户组合进行压力测试；
5. 在生成客户材料前，完成适当性、收益承诺与风险提示检查；
6. 经研究员、投资经理或财富顾问确认后，输出最终结果。

目前，上述流程主要通过产品原型、架构设计与示例文档进行展示，尚未开发为可供金融机构部署的完整系统，也未接入金仕达或其他金融机构的生产环境。

## 演示任务

为完整展示工作流程，我们选取了一个范围相对明确的宏观分析场景：

> **美联储释放进一步加息信号后**，分析黄金、原油、美股与人民币资产可能受到的影响，并查看一个模拟客户组合在不同情景下的风险变化。

该场景能够一次性串联原文检索、因果分析、情景推演、组合压力测试、客户材料生成与合规审核等多个环节。

> ⚠️ 页面中的客户信息、持仓数据、行情数据与压力测试结果**均为模拟数据**，仅用于展示系统流程，不构成任何投资建议。

- **金融信息终端与交易工具** — Trader Terminal v0.1 已完成；财经资讯聚合管线持续运行；TheFirstTrader 仍处于开发阶段。

## 项目边界

为避免误解，特此声明本项目的边界：

- 不提供自动荐股或自动投资决策；
- 不承诺任何收益，也不替代持牌投顾、投资经理或其他金融专业人士；
- 所有客户信息、持仓数据、行情数据及压力测试结果均为模拟数据；
- 尚未接入金仕达或其他金融机构的生产系统；
- 评测指标需在 MVP 完成后通过正式对照实验验证；
- 公开仓库不包含任何 API 密钥、账户信息、服务器地址、真实交易记录或其他敏感数据。
