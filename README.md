## 一、项目背景
金融市场中的行情、公告、财报、政策、新闻和研究报告数量较多，但这些资料通常分散在不同的平台中。
在完成一项投资研究任务时，使用者往往需要依次进行资料检索、原文核对、数据整理、逻辑分析、组合影响评估和报告撰写。不同环节由不同工具承担，信息和结论难以连续传递。
通用大模型可以提高资料整理和文本生成效率，但直接用于金融研究时，仍可能出现以下问题：
- 未明确说明数据截止时间；
- 关键数字缺少原始出处；
- 事实、市场观点和模型判断混杂；
- 推断条件和不确定性表达不足；
- 研究结论无法直接对应客户持仓和风险约束；
- 生成的客户材料缺少适当性和风险提示检查。
金策希望在金融资料检索、研究分析、组合评估和客户材料生成之间建立较为完整的工作流程。
项目不以直接回答“买什么”为目标，也不代替研究员、投资经理、财富顾问或其他专业人员作出决定。

## 二、项目方案
金策围绕一项具体研究任务组织工作流程：
1. 明确研究范围、数据截止时间和可使用的信息来源；
2. 将资料检索、数据处理、宏观分析、情景分析等任务分别执行；
3. 对重要结论保留原始出处、发布时间、关键假设和失效条件；
4. 区分已确认事实、市场观点、分析推断和情景假设；
5. 将研究结论映射到模拟客户组合，进行风险暴露分析和情景压力测试；
6. 在生成客户材料前检查适当性、收益承诺、风险提示和依据完整性；
7. 由研究员、投资经理或财富顾问确认后输出最终材料。
当前方案主要通过研究文档、流程图、HTML原型和报告样稿进行展示，尚未形成可供金融机构正式使用的完整系统。

## 三、演示任务
项目首期选择以下宏观研究任务作为原型场景：
> 美联储释放进一步加息信号后，分析黄金、原油、美股和人民币资产可能受到的影响，并查看一个模拟客户组合在不同情景下的风险变化。
该场景用于展示以下环节：
- 原始资料检索和时间核对；
- 宏观变量传导关系整理；
- 基准、乐观和悲观情景设定；
- 不同资产可能受到的影响；
- 模拟组合风险暴露分析；
- 内部研究材料生成；
- 客户沟通材料生成；
- 合规检查和人工确认。
原型中的客户信息、持仓、行情和压力测试数字均为模拟数据，仅用于产品展示，不构成投资建议。

## 四、主要功能设计
### 1. 研究任务管理
记录研究问题、任务范围、数据截止时间、分析对象和输出要求。
### 2. 多来源资料整理
汇总政策文件、新闻、市场数据、公告、财报和研究资料，并保留来源及发布时间。
### 3. 分工式研究流程
将研究规划、资料检索、数据分析、宏观分析、情景分析、组合分析和材料整理分开处理。
### 4. 结论依据记录
为重要结论保留原始出处、关键数据、分析假设、置信程度和失效条件。
### 5. 情景与组合分析
将研究结论放入模拟客户组合，比较不同市场情景下的风险暴露和可能变化。
### 6. 客户材料整理
根据内部研究结果，形成适合客户阅读的说明材料，同时保留必要的风险提示。
### 7. 合规检查
检查收益承诺、适当性冲突、依据不足、风险提示缺失及其他不恰当表述。
### 8. 人工确认
系统输出作为研究和材料整理的辅助结果，最终内容由相关专业人员确认。

## 八、评测设计
项目计划比较以下三类方案：
1. 普通单模型；
2. 单模型加检索增强；
3. 金策的分工研究、证据记录和审核流程。
拟评估的指标包括：
- 事实准确率；
- 数字准确率；
- 引用覆盖率；
- 引用对结论的支持程度；
- 无依据结论比例；
- 数据时间识别准确率；
- 组合计算准确率；
- 合规风险识别率；
- 任务完成时间；
- 系统调用成本；
- 人工修改量。
目前仅完成评测框架和测试目标设计，尚未开展正式实验。
