金策 FinSight 行业与竞品研究

版本 v1.1 | 2026-07-19 事实分层标注：【官方事实】【监管文件】【行业报告】【学术研究】【用户观点】【推断】【待核实】 本文由《04_研究底稿》整理而来。能查到发布主体或文号的，按事实引用；公开页面没有披露的，不用“没有该能力”代替；【待核实】条目不进入正式判断。

【第1页】

摘要

这次研究先回答四个比较实际的问题：市场量级是否足够大，金融机构是否真的开始使用大模型，现有工具具体卡在哪里，以及金策能不能找到一个不过度依赖数据资源的位置。

  1. 市场量级可以确认。 中国个人可投资资产2022年末为278万亿元【行业报告：招商银行×贝恩】。麦肯锡采用另一口径，也将中国列为全球第二大财富管理市场【行业报告】。
  2. 机构采用已经发生。 HithinkGPT完成网信办备案；2025年2月，至少16家券商完成DeepSeek-R1/V3本地化部署【官方事实】。这至少说明技术已进入实际业务环境，而不只是产品发布会。
  3. 可靠性仍是使用障碍。 FinanceBench测试中，GPT-4-Turbo结合RAG在150道金融问答中有81%答错或拒答【学术研究】。但这是一项特定基准测试，不能直接等同于所有金融AI产品的真实错误率。
  4. 金策可先解决具体流程问题。 与其和Bloomberg、Wind比数据覆盖，不如把来源定位、事实与观点区分、组合影响说明、客户材料检查和人工确认放在同一工作流中【推断】。

目前证据能支持“市场大、机构已采用、可靠性和监管要求重要”。它还不能证明机构一定会采购金策，也不能证明竞品没有类似功能。这两点需要后续访谈和产品实测。

【第2页】

1. 行业现状（一）：财富管理市场规模

1.1 我先确认了市场量级

  • 中国个人可投资资产2022年末达278万亿元【行业报告：招商银行×贝恩《2023中国私人财富报告》】。
  • 麦肯锡估算，2020年底中国个人金融资产约200万亿元，居全球第二；预计2025年达332万亿元【行业报告】。

两组数字不能直接拼接成增长曲线，因为“可投资资产”和“个人金融资产”的统计口径不同。它们共同能说明的是：中国财富管理市场体量很大，并且报告均判断其仍会增长。

1.2 这与工具需求有什么关系

【推断】资产和客户数量增加后，研究、配置、风险检查和客户沟通的工作量通常也会增加。机构是否因此增加智能投研预算，还受组织模式、数据基础和采购周期影响，不能仅凭宏观资产规模直接得出结论。

1.3 证据边界

市场总量来自行业报告，可信度较高；“智能投研工具市场会同步增长”目前只有方向性依据。底稿没有该细分市场的公开规模数据，因此本文不估算市场收入或增速。

【第3页】

1. 行业现状（二）：券商研究业务与买方投顾转型

1.4 已有报告怎么描述转型

  • 麦肯锡提出，券商财富管理转型中“三分投、七分顾”已成行业共识【行业报告】。“投”涉及研究与配置，“顾”还包括客户解释、持续服务和适当性要求。
  • 基金投顾试点于2019年10月启动，约60家机构获资格【行业报告】。这一数量的最新情况没有在本轮检索中核实，因此只保留为背景，不用于判断当前竞争规模。

1.5 我更关注中间的手工步骤

【推断】研究报告并不能直接变成面向某位客户的材料。投资经理需要结合组合判断影响，财富顾问还要结合客户风险等级改写表达、补充风险提示并送审。读报告、查数字、对持仓、写说明、过合规，往往分散在不同系统和人员之间。

这不等于所有机构都存在同样的问题。更稳妥的说法是：金策应优先验证这些转交步骤是否重复、是否容易丢失来源，以及哪一类人员最愿意为改进付费。

1.6 可能涉及的使用者

  • 研究员：查找原文、核对数字、标记假设。
  • 投资经理：判断事件或观点对组合的影响。
  • 财富顾问：把研究内容改写成适合客户阅读的材料。
  • 风控合规人员：检查适当性、风险提示和留痕。

这四类角色与金策的目标用户相符，但角色匹配只是产品设计依据，不是需求已经验证的证据【推断】。

2.2 这组事实能说明什么

第一，金融大模型已有备案产品，也已有券商本地化部署案例。第二，公开披露的用途主要是检索、文档、研究和问答，说明机构更愿意先让模型辅助人员，而不是让模型直接作投资决定。

“本地化部署是所有机构的默认选择”证据还不够。至少16家券商的案例能说明它是重要方案，但不能代表全部金融机构【推断】。

【第5页】

2. 市场趋势（二）：从“能回答”到“回答后能否使用”

产品发布数量已经不少。真正落到研究工作中，我认为还要继续问三个问题：

  1. 输出数字能否定位到财报科目、公告原文或数据字段；
  2. 结论修改过几次、由谁确认，是否能够回看；
  3. 面向客户的内容是否检查风险提示、适当性和AI内容标识。

2.3 这里证据还不够强

从公开页面很难完整判断各厂商的内部版本。一些产品会展示AI问答或引用功能，但部署方式、审计记录粒度、适当性检查等通常没有公开披露。因此，本文只能说这些能力值得比较，不能说行业“普遍没有解决”【推断】。

2.4 对金策设计的启发

【推断】如果底座模型能力逐渐接近，产品差异会更多体现在：答案能否回到原文、事实和观点是否分开、组合影响如何解释、客户材料在发出前检查了什么。金策应把这些操作做得可见、可检查，而不是把“大模型、RAG、多智能体”本身当作卖点。

【第6页】

3. 用户痛点（一）：检索、核验与数字引用

3.1 能找到的量化证据

  • 麦肯锡全球研究院称，知识型员工约19%的工时用于检索搜集信息【行业报告，非投研专属，证据强度中】。
  • 投研工作确实涉及行情、财报、公告、政策和新闻等多种来源，但本轮没有找到“投研人员检索工时占比”的公开定量研究。不能把19%直接改写成投研行业数据。

3.2 为什么生成快了，核验未必更快

研究材料中的数字通常需要回到原始出处。若AI只给出一个数字和模糊来源，使用者仍要打开财报或公告重查。此时节省的是写句子的时间，新增的是核验答案的时间。

“数字不敢引用”是公开从业讨论中常见的说法【用户观点，未逐帖取证】，不是本项目完成访谈或问卷后得到的调研结果。FinanceBench能从侧面说明财报问答确有错误风险，但不能证明所有从业者都把它视为首要痛点。

3.3 暂时的产品判断

【推断】金策若要解决这个问题，最小可验证功能不是“自动写完整报告”，而是让关键数字带上原文位置、口径、时间和人工确认状态。

【第7页】

3. 用户痛点（二）：研究内容如何走到组合和客户材料

3.4 信息在交接中可能丢失

【推断】一种常见作业路径是：研究员形成观点，投资经理结合持仓判断影响，财富顾问再转成客户能理解的表达。若系统之间只传递最终文字，前面的数据来源、假设和失效条件可能不会一起传下去。

这里没有机构样本或流程耗时数据，所以不能写成全行业确定存在的“断层”。后续更适合用真实任务观察：同一条研究结论经过三类角色后，哪些信息被删掉、重复核验或重新录入。

3.5 合规检查通常发生在哪一步

适当性管理要求了解投资者、进行产品分级、把适当产品提供给适当投资者，并完成风险揭示留痕【监管文件：证监会令第130号】。至于机构实际采用事前还是事后审查，本轮没有流程调研，不能一概而论。

金策可以验证的具体问题是：在生成客户材料时同步检查风险等级、收益承诺和风险提示，是否比成稿后再修改更少返工【推断】。

【第8页】

4. 金融机构AI应用挑战（一）：幻觉与数字错误

4.1 FinanceBench提供了一项较强的警示证据

Patronus AI发布的FinanceBench基准（arXiv:2311.11944）测试显示：在150道基于上市公司财报的金融问答中，GPT-4-Turbo结合RAG有81%答错或拒答【学术研究】；论文归因包括模型幻觉和无法从长篇财报中正确取证。

这个数字需要按原实验范围理解。它说明特定模型、特定RAG方案在该测试集上表现不可靠，不代表所有模型、所有配置或所有金融任务都有81%的错误率。

4.2 容易出问题的地方

底稿归纳出四类风险【学术研究/底稿归纳】：

  • 数字计算或转述错误；
  • 使用过期数据；
  • 给出结论却没有可定位来源；
  • 把观点、传闻或推断写成事实。

4.3 对产品的直接要求

【推断】金融机构不能只看答案是否流畅，还要知道答案引用了什么、数字如何计算、信息截至何时。即使提供来源，也仍需人工抽查；产品的价值是缩小核验范围，而不是宣称消除错误。

【第9页】

4. 金融机构AI应用挑战（二）：机构对AI输出承担什么责任

4.4 Air Canada案能提供的提醒

在Air Canada案（2024 BCCRT 149）中，该公司客服机器人编造了不存在的退款政策，仲裁庭裁定公司必须兑现机器人所作承诺【裁决文书】。至少在该案中，“由机器人生成”没有免除公司的责任。

4.5 不能把境外客服案例直接当成中国金融判例

该案不是金融案例，也不适用中国法律框架。本文引用它，只用于说明一个治理问题：当机构把AI输出提供给客户时，不能假设责任会转移给模型供应商【推断】。

中国金融场景还要分别遵守适当性、数据安全、生成式AI服务和特许业务等要求【监管文件/官方事实】。因此，客户材料中的风险提示、适用对象、内容标识和人工确认都需要留下记录。

4.6 我得到的产品判断

【推断】金策应让使用者在发出材料前看到来源、数据时间、风险提示和检查结果。它不能代替机构合规判断，也不能以“系统已检查”承诺输出必然合法。

【第10页】

5. 竞品分析（一）：Bloomberg / Wind

这一页先看两类成熟终端。它们的数据积累和机构覆盖是金策短期无法复制的，比较重点应放在AI输出如何引用、部署方式以及研究内容如何进入客户材料。

5.1 Bloomberg（彭博终端）

  • 目标用户：全球机构交易员、研究员、投资经理。
  • 数据覆盖：全球市场、固收、宏观和新闻，底稿将其归为覆盖最全的终端。
  • AI能力：自研BloombergGPT，500亿参数（arXiv:2303.17564）【学术研究】。
  • 组合分析：终端内含成熟组合与风险分析；财富管理以机构端为主。
  • 部署：底稿记录为不提供私有化。
  • 价格：底稿记录的第三方公开报价为27,660美元/用户/年，仅作量级参考；合同价格和当前价格仍需询价核实。
  • 公开讨论中常见说法【用户观点，未逐帖取证】：数据覆盖和功能深度受到认可；价格高、界面陈旧、学习成本高也经常被提到。这不是抽样调查结果。
  • AI回答能否逐句溯源，公开资料不足【待核实】。

5.2 Wind（万得）

  • 目标用户：中国机构研究员、投资经理和风控人员。
  • 数据覆盖：中国本土数据是其主要优势。
  • AI能力：已上线Wind Alice助手与WindClaw智能体平台【官方事实】。
  • 商业模式：机构订阅、模块化收费；本轮未取得官方现行价格。
  • 公开讨论中常见说法【用户观点，未逐帖取证】：本土数据和机构覆盖常获认可；涨价、模块收费复杂和Mac支持问题也被提及。不能据此推断全部用户体验。
  • AI引用粒度、组合与财富模块边界、私有化方式，官方公开页面未充分披露【待核实】。

【第11页】

5. 竞品分析（二）：同花顺iFinD / 东方财富Choice

5.3 同花顺iFinD

  • 目标用户：国内机构研究人员及个人专业投资者。
  • 数据覆盖：国内股、债、基金、宏观和资讯；问财较早落地自然语言查询。
  • AI能力：HithinkGPT为业内首家通过网信办备案的金融对话大模型【官方事实】。
  • 商业模式：订阅制。底稿仅记录其价格低于Wind，缺少官方报价，因此不用于采购成本结论。
  • 公开讨论中常见说法【用户观点，未逐帖取证】：自然语言查询和性价比被提及较多；也有人讨论数据准确度和销售打扰。后两项没有逐帖核验或样本统计。
  • 组合分析、财富管理和私有化细节【待核实】。

5.4 东方财富Choice

  • 目标用户：国内机构及个人投资者，依托东方财富生态。
  • 数据覆盖：行情、资讯、研报和基金数据；零售端产品连接较多。
  • AI能力：“妙想”金融大模型已接入终端【官方事实】。
  • 商业模式：订阅制、分层版本；本轮未取得官方现行价格。
  • 公开讨论中常见说法【用户观点，未逐帖取证】：生态联动和零售服务链路是常见正面评价；“专业深度不及Wind”“稳定性一般”只是部分公开讨论中的观点，不能写成产品测评结论。
  • 组合分析和私有化细节【待核实】。

我暂时不会据此判断iFinD或Choice“不重视机构级可信工作流”。公开资料不足时，更合理的下一步是产品试用和采购访谈。

【第12页】

5. 竞品分析（三）：Koyfin / AlphaSense / Fiscal.ai

海外新工具提供了几个可观察的产品做法：句级引用、字段级溯源，以及直接为投顾生成客户报告。它们能说明这些功能已被产品化，但不能自动证明同一方案适合中国机构。

5.5 Koyfin

  • 目标用户：海外买方分析师、投顾（RIA）。
  • 数据覆盖：8万+全球公司。
  • AI与投顾功能：AI助手；Advisor版提供模型组合和客户报告。
  • 价格：底稿记录的第三方公开报价为39—299美元/月，仅作版本量级参考，当前价格和附加数据费用需以官方报价为准。
  • 公开讨论中常见说法【用户观点，未逐帖取证】：可视化和相对Bloomberg的价格受到认可；固收数据较弱、实时行情需加价也被提及。

5.6 AlphaSense

  • 目标用户：企业战略、咨询和买方研究。
  • 数据覆盖：5亿+文档。
  • AI能力：Deep Research等研究功能；句级引用是公开卖点。
  • 部署：Enterprise版提供私有云。
  • 商业模式：企业订阅，本轮未取得官方现行报价。
  • 公开讨论中常见说法【用户观点，未逐帖取证】：引用定位和检索质量受到认可；价格高、内容分级收费也经常被讨论。

5.7 Fiscal.ai（原FinChat）

  • 目标用户：基本面个人投资者与小型机构。

以下均为【推断】，不是金仕达已确认的合作方案：

9. 金策的差异化定位（一）：先避开不能赢的比较

三类产品各自已经解决了一部分问题：

  • 终端类（Bloomberg、Wind、iFinD、Choice）强在数据和行情。公开页面对AI引用粒度、适当性检查和客户材料流程披露不一，不能据此判断它们没有相关能力。
  • 搜索/文档AI类（AlphaSense、Fiscal.ai）已经把句级引用或字段级溯源做成明确功能；但中国本土数据、组合分析和中国监管适配不是其公开重点。
  • 通用大模型（包括机构本地化部署的DeepSeek）生成和推理能力较强，但FinanceBench说明，直接处理财报问答仍可能出错【学术研究】。机构需要另外接数据、权限、引用和检查流程。

金策可以先做什么

【推断】金策不应自称新的数据终端，也不应承诺代替投顾。更具体的定位是：把研究原文和数字出处带到组合影响说明，再把可使用的内容转成经过风险提示、适当性和人工确认的客户材料。

产品边界：不自动荐股，不承诺收益，不替代持牌投顾；重要内容显示来源、数据时间、风险提示和确认记录。模型只负责辅助，最终判断由机构人员作出。
