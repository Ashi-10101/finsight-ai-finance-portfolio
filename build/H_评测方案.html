金策 FinSight MVP 评测方案（H）

状态说明：金策 FinSight 当前处于方案设计 + 产品原型阶段（未开发完成，未上线）。本文件中全部目标值均为 MVP目标值，待后续对照实验验证；所有评测数字在实验完成前不得对外引用为既有成果。

1. 评测目标

验证金策 FinSight 的核心假设：相对于"普通单模型"与"单模型 + RAG"基线，"多智能体 + 证据链 + 合规即流程"架构能在事实可信度、证据可追溯性、组合计算正确性、合规召回与端到端效率上取得可测量的、统计显著的改进，且成本与延迟在可接受范围内。

2. 指标体系（14 项）

每项指标统一给出：定义 / 计算方法 / MVP目标值（全部标注"MVP目标值，待后续对照实验验证"）。

2.1 事实准确率（Factual Accuracy）

  • 定义：输出中陈述型事实（事件、政策、主体行为）与权威来源一致的比例。
  • 计算方法：程序化校验将抽取的事实陈述与标准答案库比对；抽样 20% 由双盲人工复核，取两者一致判定。事实准确率 = 正确事实数 / 事实陈述总数。
  • MVP目标值：≥ 95%（MVP目标值，待后续对照实验验证）。

2.2 数字准确率（Numeric Accuracy）

  • 定义：输出中的数值（价格、涨跌幅、比率、财务数据）与来源数值一致的比例，含单位与口径。
  • 计算方法：正则 + 结构化抽取全部数字，回溯至来源字段逐项比对，容差为来源更新造成的 ±0.5%。数字准确率 = 正确数字数 / 数字总数。
  • MVP目标值：≥ 98%（MVP目标值，待后续对照实验验证）。

2.3 引用覆盖率（Citation Coverage）

  • 定义：应附证据的结论/事实中，实际携带有效证据编号的比例。
  • 计算方法：人工标注每份输出"需要引用的陈述点"清单，程序统计其中带 [E] 编号的比例。引用覆盖率 = 带证据陈述点数 / 应引用陈述点总数。
  • MVP目标值：≥ 95%（MVP目标值，待后续对照实验验证）。

2.4 引用支持率（Citation Support Rate）

  • 定义：已给出的引用中，原文内容确实支持对应陈述的比例（防止"引了但引错/引无关"）。
  • 计算方法：抽样全部引用中的 30%（且不少于 50 条），双盲人工判断"支持 / 部分支持 / 不支持"；程序化用原文片段与陈述的蕴含校验辅助。引用支持率 =（支持 + 0.5×部分支持）/ 抽样引用数。
  • MVP目标值：≥ 90%（MVP目标值，待后续对照实验验证）。

2.5 无依据结论比例（Unsupported Conclusion Rate）

  • 定义：结论性、建议性陈述中，无法由任何已引用证据或显式标注的假设/推断支撑的比例。越低越好。
  • 计算方法：人工标注每条结论的支撑状态（有证据 / 有标注假设 / 无依据）。无依据结论比例 = 无依据结论数 / 结论总数。
  • MVP目标值：≤ 5%（MVP目标值，待后续对照实验验证）。

2.6 信息时效性（Information Timeliness）

  • 定义：输出使用的信息与数据在其标注时点仍然有效、未引用已过期或已被更正信息的比例。
  • 计算方法：对每条证据核对"发布时点 / 更新时点 / 任务截止时点"，判定是否过期（如引用已被修正的初值数据）。时效性 = 时效合格证据数 / 证据总数；同时记录证据时间戳标注完整率。
  • MVP目标值：时效合格率 ≥ 95%，时间戳标注完整率 = 100%（MVP目标值，待后续对照实验验证）。

2.7 组合计算准确率（Portfolio Calculation Accuracy）

  • 定义：集中度、敞口、压力测试损益、风险等级匹配等组合计算结果与程序化重算基准一致的比例。
  • 计算方法：用独立实现的计算脚本（双重实现）对每个任务重算标准值，与系统输出逐项比对，容差为四舍五入误差。组合计算准确率 = 计算正确项数 / 计算项总数。
  • MVP目标值：= 100%（确定性计算不允许出错；MVP目标值，待后续对照实验验证）。

2.8 合规风险识别召回率（Compliance Recall）

  • 定义：测试集中预埋的合规风险点（收益承诺话术、无依据投资建议、适当性不匹配、风险提示缺失、AI 标识缺失等）被系统检出的比例。
  • 计算方法：预埋风险点清单作为标准答案，统计召回率 = 检出风险点数 / 预埋风险点总数；同时记录精确率（检出中真实为风险的比例）。
  • MVP目标值：召回率 ≥ 90%，精确率 ≥ 85%（MVP目标值，待后续对照实验验证）。

2.9 报告生成时间（Report Generation Time）

  • 定义：从任务提交到生成可送人工确认的内部版报告草稿的端到端墙钟时间。
  • 计算方法：记录提交时间戳与草稿完成时间戳之差，按任务类型分层统计 P50 / P95。
  • MVP目标值：单事件类任务 P50 ≤ 10 分钟，P95 ≤ 20 分钟（对照组人工基线预计 2–4 小时；MVP目标值，待后续对照实验验证）。

2.10 用户任务完成率（Task Completion Rate）

  • 定义：用户（研究员/财富顾问角色扮演者）在可用性测试中无需系统外补救即可完成规定任务的比例。
  • 计算方法：招募 5–8 名具金融背景的受试者执行标准任务脚本，记录"独立完成 / 需外部补救 / 失败"。完成率 = 独立完成任务数 / 任务总数。
  • MVP目标值：≥ 85%（MVP目标值，待后续对照实验验证）。

2.11 模型成本（Model Cost）

  • 定义：单任务端到端完成所产生的模型 API 成本（含多智能体全部调用与 repair 轮次）。
  • 计算方法：按调用记录（token 数 × 各模型单价）汇总，区分规划/检索/分析/审核/整合各阶段成本。
  • MVP目标值：单任务 ≤ 10 元人民币等值（MVP目标值，待后续对照实验验证；最终以实验时实际单价折算）。

2.12 系统延迟（System Latency）

  • 定义：交互过程中的关键响应延迟，包括首 token 延迟与阶段间流转延迟（区别于 2.9 的端到端总时长）。
  • 计算方法：SSE trace 落库时间戳计算：首字节延迟 P50/P95、各智能体阶段延迟分布。
  • MVP目标值：首字节延迟 P50 ≤ 5 秒；单阶段流转 P95 ≤ 60 秒（MVP目标值，待后续对照实验验证）。

2.13 失败率（Failure Rate）

  • 定义：任务因系统错误（编排异常、检索失败、输出格式不合规、超时）未能产出可评审结果的比例。
  • 计算方法：失败任务数 / 总任务数；按失败类型分类统计，人工确认失败归因。
  • MVP目标值：≤ 5%，且全部失败可追溯至日志（MVP目标值，待后续对照实验验证）。

2.14 用户满意度（User Satisfaction）

  • 定义：受试者对输出可信度、可解释性、可用性的主观评价。
  • 计算方法：任务后问卷，采用 SUS 量表（10 题，0–100）+ 3 个领域定制题（证据可追溯性、分层表达清晰度、合规放心度，5 点李克特）。
  • MVP目标值：SUS ≥ 75，定制题均分 ≥ 4.0/5（MVP目标值，待后续对照实验验证）。

3. 三组对照实验设计

3.1 实验组设置

组 配置 说明
A 普通单模型 单一通用大模型，直接回答任务，无检索、无证据链、无合规模块
B 单模型 + RAG 同一基础模型 + 向量检索增强，可访问与 C 组相同的语料库，但无多智能体分工、无证据链字段约束、无合规审查
C 金策方案 多智能体投研引擎 + 可信证据链 + 因果推演 + 组合计算 + 合规即流程

控制变量：三组使用同一基础模型（同版本、同温度）、同一语料库、同一任务集、同一提示词工程预算（A/B 组提示词经同等程度的优化，避免"故意弱化基线"）；B、C 组检索语料与时间点一致。

3.2 统一测试集（≥ 30 个金融研究任务）

  • 宏观事件类（10 个）：如"美联储释放进一步加息信号，分析对黄金、原油、美股、人民币资产的影响""非农数据超预期后的利率路径重估"等；要求事实清单、因果链、三情景推演。
  • 公司基本面类（10 个）：如"基于公开财报与公告分析某公司毛利率变动原因""某公司重大事项公告的事实核查与影响分析"等；要求数字准确、引用可下钻。
  • 组合与合规类（10 个）：基于模拟持仓（明确标注模拟数据）的集中度/敞口/压力测试计算、风险等级适当性匹配、含预埋合规风险点（收益承诺话术、无依据建议、风险提示缺失）的客户沟通材料改写任务。

任务设计原则：每个任务配标准答案要点与评分细则；预埋"陷阱"（过期数据、易混淆口径、单位换算、相关性≠因果）；难度参照公开基准校准。

3.3 评审方式

  • 程序化校验：数字比对、组合计算重算、引用编号解析、时间戳校验、合规关键词与话术规则初筛、日志完整性检查。全自动、可复现。
  • 人工双盲评审：≥ 2 名具金融背景的评审人（不告知组别、随机打乱顺序）按统一 rubric 评分；评分维度含事实正确性、论证质量、证据有效性、分层表达、合规完备性。评审人间一致性用 Cohen's κ 度量，κ < 0.6 的维度须仲裁重评。

3.4 基线参考与任务难度佐证

公开基准 FinanceBench（arXiv:2311.11944）显示，GPT-4-Turbo + RAG 配置下金融问答约 81% 的回答答错或拒答，说明金融研究任务对通用模型具有高难度、幻觉风险高。本测试集在发布前将抽样若干任务在该基准同类题目上校准难度，确保任务区分度（A 组得分显著低于满分、C 组目标显著高于 A/B）。

4. 数据采集与统计方法

4.1 数据采集

  • 系统侧：运行日志（runs/run_steps 落库）、token 用量、各阶段时间戳、失败堆栈、检索命中记录、合规审查记录、人工确认记录——全部自动采集，禁止事后手工填报。
  • 评审侧：程序化校验结果入库；人工评分表结构化录入，保留双人原始分与仲裁记录。
  • 用户侧：任务完成观察记录 + SUS/李克特问卷，受试者签署知情同意。

4.2 统计与显著性检验

  • 主要比较：C vs B、C vs A 在事实准确率、引用支持率、合规召回率等核心指标上的差异。
  • 比例类指标（准确率、召回率、完成率）：卡方检验或 Fisher 精确检验（小样本时）；报告 95% 置信区间（Wilson 区间）。
  • 连续类指标（生成时间、延迟、成本、SUS）：先检验分布，正态用独立样本 t 检验，非正态用 Mann-Whitney U；报告效应量（Cohen's d 或 rank-biserial）。
  • 多重比较校正：核心指标族采用 Holm-Bonferroni 校正，控制族错误率。
  • 样本量说明：30 任务为 MVP 阶段可行性规模；对预期效应量（如准确率提升 15 个百分点）做事前功效估算，功效不足时在报告中如实标注"探索性结果"。
  • 预注册：实验假设、指标定义、分析方法在实验开始前冻结存档，防止事后挑选指标。

4.3 评测伦理

  • 仅使用公开信息（央行公告、交易所公告、上市公司公开披露、公开新闻）与明确授权数据；不使用任何未经授权的付费数据与研报；语料库逐条登记来源与授权状态。
  • 所有客户画像与持仓均为模拟数据，材料中显著标注；不涉及任何真实客户信息。
  • 受试者知情同意，可自由退出；问卷数据匿名化存储。
  • 不产生真实投资建议；测试输出统一标注"非投资建议"；不对外发布任何可能被误解为荐股的内容。
  • 评测过程遵守《生成式人工智能服务管理暂行办法》（国家网信办等七部门令第15号）关于标识与安全的要求；涉及证券研究业务的表述，以《证券期货投资者适当性管理办法》（证监会令第130号）、《证券基金经营机构信息技术管理办法》（证监会令第152号）、《证券期货业网络和信息安全管理办法》（证监会令第218号）为合规参照。
  • 实验失败、负结果与成本超支如实披露；不删除对项目不利的数据。

5. 里程碑

  1. 测试集与评分细则冻结（含预注册文档）
  2. 三组环境搭建与冒烟测试（每组 3 任务）
  3. 正式实验（30 任务 × 3 组，含失败重试记录）
  4. 程序化校验 + 双盲人工评审 + 一致性仲裁
  5. 统计分析与结果报告（含负结果）
  6. 可用性测试（5–8 名受试者）与满意度问卷

本方案全部结论在实验完成前标注为"待验证"；实验完成后以实测值替换 MVP 目标值并保留目标值作为对照。
